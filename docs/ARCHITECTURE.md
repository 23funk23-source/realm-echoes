# Realm of Echoes — архитектура

## Принципы

- **Ноль зависимостей, ноль сборки**: открыл `index.html` — играешь. Canvas 2D, ванильный JS (ES2020), Web Audio API.
- Весь рендер — процедурный (фигуры на канвасе), вся музыка — синтез в рантайме. В репозитории нет бинарных ассетов.
- Один глобальный игровой цикл на `requestAnimationFrame` с дельта-временем (clamp 33 мс — защита от скачков после сворачивания вкладки).

## Структура файлов

```
index.html        — канвас + HTML-оверлеи (меню выбора класса, смерть, победа)
css/style.css     — стили оверлеев и карточек классов
js/audio.js       — Music: генеративный музыкальный движок + SFX (IIFE, глобальный объект)
js/game.js        — вся игровая логика и рендер
docs/             — документация
```

`audio.js` подключается раньше `game.js` и не зависит от него; `game.js` дергает `Music.ensure() / setBoss() / sfx() / toggleMute()`.

## game.js — устройство

### Состояния
`state ∈ menu | play | pause | dead | win`. Оверлеи (`#menu`, `#gameover`, `#victory`) — обычные HTML-блоки, переключаются классом `.hidden`. Канвас рисуется всегда.

### Сущности (plain objects, без классов)
- `player` — позиция, HP, уровень/опыт, кулдауны (выстрел, умение, неуязвимость), ссылка на конфиг класса из `CLASSES`.
- `enemies[]` — миньоны; конфиги видов в `KINDS` (slime/bat/cultist/skeleton), поведение — `chase` / `shoot` / `shoot3`.
- `boss` — активный босс или `null`; конфиги в `BOSSDEFS`, у каждого своя функция `update(b, dt)` с таймерами паттернов.
- `bullets[]` / `ebullets[]` — снаряды игрока и врагов (раздельные массивы — раздельные проверки коллизий).
- `drops[]`, `particles[]` — зелья и частицы.

### Этапы
`STAGES[]` — линейный сценарий забега (волна → босс → …). Менеджер этапов в `update()`: этап считается пройденным, когда `enemies` пуст (волна) или босс убит; далее пауза 1.6 с (`nextT`) и `startStage(i+1)` либо победа.

### Коллизии
Только круг-круг (`dist² < (r1+r2)²`), полный перебор. При масштабах игры (<100 сущностей, <300 пуль) этого достаточно — пространственные структуры не нужны.

### Камера и мир
Мир — квадрат `WORLD × WORLD` (1800 px). Камера следует за игроком с clamp по границам; если мир меньше экрана — центрируется. Тряска экрана — случайное смещение камеры с затуханием.

## audio.js — устройство

Граф: `осцилляторы → gain(огибающая) → master → lowpass → destination`, параллельно посыл в `convolver` (импульс реверберации генерируется из шума при инициализации).

- Шаговый секвенсор на `setInterval`; каждый шаг планирует ноты на `audioContext.currentTime + 0.06` (для эмбиента точности достаточно, lookahead-планировщик не нужен).
- Гармония: цикл аккордов Am–F–C–G; каждые 16 шагов — пад (синусы, атака 1.4 с), на каждом шаге — нота арпеджио по паттерну (−1 в паттерне = пауза).
- `setBoss(true)` — перезапуск интервала с меньшей длительностью шага + сатурированный бас каждые 4 шага.
- Mute — просто `master.gain = 0`, секвенсор продолжает идти (мгновенный unmute без рассинхрона).
- Инициализация контекста — лениво, по первому пользовательскому клику (`ensure()`), иначе браузер заблокирует звук.

## Визуальный слой (v1.6)

- **Земля** — процедурные тайлы 64px: для каждого видимого тайла зона определяется через `zoneAt`, шейд и деталь берутся из `TILES[zoneKey]` по перемешанному хэшу координат (без внешних текстур).
- **Спрайты** — 16×16 сетки из `sprites.js`; `outlined()` один раз строит версию 18×18 с тёмной обводкой по силуэту. Анимация — процедурная (bob + squash/stretch + наклон), без кадровых листов.
- **Свет** — аддитивный проход (`globalCompositeOperation='lighter'`) для «блума» пуль/частиц и ореола боссов; композит и `globalAlpha` восстанавливаются до отрисовки UI.

## Известные упрощения

- Нет прицельного lookahead-планировщика аудио — на фоновой вкладке `setInterval` троттлится и музыка «редеет» (приемлемо: игра на фоне не идёт).
- Нет object pooling для пуль/частиц — на текущих объёмах GC-пауз не видно.
- Баланс захардкожен в конфигах `CLASSES` / `KINDS` / `BOSSDEFS` — все числа в одном месте, править легко.
